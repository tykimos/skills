# health-care - 건강 잔디 심기 스킬 PRD

## 1. 개요

**스킬명:** health-care
**컨셉:** GitHub 잔디(contribution graph) 심기처럼, 매일의 건강 습관을 시각화하고 장려하는 AI 헬스케어 에이전트 스킬
**목표:** 사용자가 매일 운동, 식단 관리, 영양제/약 복용을 꾸준히 실천하도록 동기부여하고 추적 관리

---

## 2. 핵심 기능

### 2.1 건강 잔디 시각화 (Health Grass Graph)

GitHub 잔디처럼 날짜별 셀로 건강 실천도를 시각화한다.

| 항목 | 잔디 색상 기준 |
|------|--------------|
| 운동 | 미실천(회색) → 가벼운 운동(연초록) → 충분한 운동(진초록) |
| 식단 | 과식/폭식(빨강) → 적정(연초록) → 목표 달성(진초록) |
| 영양제/약 | 미복용(회색) → 일부 복용(연초록) → 전부 복용(진초록) |
| 종합 | 3개 항목의 가중 평균으로 종합 잔디 색상 결정 |

**시각화 포맷:**
- HTML 기반 히트맵 (최근 12주 / 6개월 / 1년 뷰)
- 터미널 텍스트 기반 간이 잔디 (일상 대화 시 인라인 표시)
- PNG 이미지 내보내기 (공유용)

### 2.2 일일 3회 메시지 시스템

| 시점 | 메시지 유형 | 내용 |
|------|-----------|------|
| 아침 (08:00) | 모닝 브리핑 | 오늘의 목표 리마인드, 어제 잔디 현황, 아침 영양제 복용 체크 |
| 점심 (12:30) | 미드데이 체크 | 점심 식사 기록 유도, 오전 운동 여부 확인, 수분 섭취 리마인드 |
| 저녁 (20:00) | 데일리 리뷰 | 하루 종합 리뷰, 잔디 업데이트 시각화, 내일 목표 설정, 저녁 영양제 체크 |

**메시지 톤:**
- 강압적이지 않고 따뜻하게 격려
- 연속 달성 시 스트릭(streak) 축하
- 놓친 날이 있어도 비난하지 않고 다시 시작 응원

### 2.3 음식/운동 사진 대응

**음식 사진 업로드 시:**
- 음식 종류 인식 및 예상 칼로리 추정
- 영양소 구성 간단 분석 (탄수화물/단백질/지방)
- 목표 대비 피드백 ("단백질이 풍부한 좋은 선택이네요!")
- 자동으로 식단 기록에 반영

**운동 사진/스크린샷 업로드 시:**
- 운동 종류 인식 (헬스, 러닝앱 스크린샷, 요가 등)
- 예상 소모 칼로리 추정
- 격려 메시지 및 잔디 업데이트
- 자동으로 운동 기록에 반영

### 2.4 데이터 기록 및 이력 관리

사용자의 건강 데이터를 메모리 파일로 영속 저장하여 세션 간 연속성 유지.

- 카테고리별 또는 종합 잔디 이미지
- SNS 공유용 카드 이미지 생성

## 5. 스킬 동작 흐름

### 5.1 최초 설정 (Onboarding)

```
사용자: /health-care 시작

에이전트:
1. 환영 메시지 출력
2. 기본 정보 수집 (인터뷰 형식)
   - 키/몸무게/목표체중
   - 하루 목표 칼로리
   - 운동 목표 (분/일)
   - 복용 중인 영양제/약 목록
   - 메시지 수신 선호 시간
3. profile.json 생성
4. supplements.json 생성
5. 첫 잔디 시각화 출력
```

### 5.2 일일 기록 흐름


## 7. 잔디 점수 산정 기준

### 0~4 스케일

| 점수 | 운동 | 식단 | 영양제/약 |
|------|------|------|----------|
| 0 | 미기록 | 미기록 | 미기록 |
| 1 | 10분 미만 | 목표 150% 초과 | 25% 미만 복용 |
| 2 | 10~20분 | 목표 120~150% | 25~50% 복용 |
| 3 | 20~30분 | 목표 100~120% | 50~75% 복용 |
| 4 | 30분 이상 | 목표 이내 | 75% 이상 복용 |

**종합 점수:** `(운동×0.4 + 식단×0.35 + 영양제×0.25)` → 0~4 스케일

## 9. 기술 구현

### 9.1 파일 구조

```
assicare/
├── prd.md                          # 이 문서
├── skill.md                        # Claude Code 스킬 정의
├── scripts/
│   ├── generate_grass.py           # HTML 잔디 히트맵 생성
│   ├── generate_grass_image.py     # PNG 잔디 이미지 생성 (matplotlib)
│   ├── generate_weight_chart.py    # 체중 변화 그래프
│   └── generate_weekly_report.py   # 주간 리포트 생성
├── templates/
│   └── grass_heatmap.html          # 잔디 히트맵 HTML 템플릿
└── examples/
    └── sample_data.jsonl           # 샘플 데이터
```

### 9.2 의존성

- Python 3.9+
- matplotlib (잔디 이미지 생성)
- Jinja2 (HTML 템플릿)

### 9.3 메모리 파일 읽기/쓰기

스킬은 Claude Code의 메모리 시스템을 활용:
- **읽기:** `Read` 툴로 메모리 디렉토리의 JSON/JSONL 파일 읽기
- **쓰기:** `Write`/`Edit` 툴로 기록 추가 및 프로필 업데이트
- **검색:** `Grep` 툴로 과거 기록 검색

### 9.4 사진 분석

Claude의 멀티모달 기능 활용:
- `Read` 툴로 이미지 파일 읽기 (Claude Vision)
- 음식 종류 및 양 추정 → 칼로리 산출
- 운동앱 스크린샷 OCR → 거리/시간/칼로리 추출

## 10. 일일 메시지 구현 방식

Claude Code의 `/loop` 스킬과 연동하여 주기적 메시지 발송:

```
/loop 4h /health-care 체크
```

또는 cron 기반으로 하루 3회 특정 시간에 메시지:

```bash
# crontab
0 8 * * * claude -p "/health-care 아침체크"
30 12 * * * claude -p "/health-care 점심체크"
0 20 * * * claude -p "/health-care 저녁체크"
```

## 11. 확장 고려사항

- **Apple Health / Google Fit 연동:** 자동 운동 데이터 수집 (향후)
